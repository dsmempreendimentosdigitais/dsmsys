Semana 1 — JavaScript Básico

A linguagem do mercado. Toda vaga remote-ready para o Canadá exige JS sólido. Esta semana cobre os fundamentos que aparecem em toda entrevista.

📅 Duração: 6 dias · 1h cada
🎯 Foco: Variáveis, tipos, funções, loops
🇬🇧 Tarefa de inglês: Nomeie todas variáveis e funções em inglês. Pense em inglês ao nomear: userList, totalPrice, getUserById.
✓DiaAulaO que fazer
1
Variáveis & Tipos
letconsttypeof
Diferença entre var, let e const. Por que nunca usar var em código moderno.
1Declare variáveis com let e const. Tente reatribuir uma const — veja o erro.
2Teste os 7 tipos primitivos: string, number, boolean, null, undefined, symbol, bigint
3Use typeof em cada variável. Por que typeof null === 'object'? (bug histórico do JS)
4Template literals: `Hello, ${name}! You are ${age} years old.`
2
Condicionais
if/elseswitchternário
Controle de fluxo. Aparece em todo algoritmo de entrevista.
1Escreva uma função getGrade(score) que retorna "A", "B", "C", "D" ou "F"
2Reescreva usando switch. Quando usar if vs switch?
3Ternário: const result = score >= 60 ? 'Pass' : 'Fail'
4Entenda truthy/falsy: 0, "", null, undefined, NaN são falsy
3
Funções
functionreturnscope
Declaração, parâmetros, retorno e escopo.
1Crie 5 funções: add(a,b), isEven(n), greet(name), celsiusToFahrenheit(c), maxOfThree(a,b,c)
2Entenda escopo: variável declarada dentro da função não existe fora. Teste isso.
3Toda função deve ter um propósito claro e nome descritivo em inglês
4
Arrow Functions
const fn = () =>
Sintaxe mais curta. É a forma padrão usada em React e Node.js modernos.
1Sintaxe básica: const add = (a, b) => a + b — retorno implícito
2Com corpo: const greet = (name) => { const msg = `Hello, ${name}`; return msg }
3Reescreva todas as 5 funções do dia 3 usando arrow functions
5
Loops
forwhilefor...of
Repetição controlada. A base de qualquer algoritmo.
1for: Some todos os números de 1 a 100. Resposta: 5050
2while: Imprima todos os números pares de 1 a 50
3for...of: Crie um array de 5 nomes e imprima "Hello, [nome]!"
6
Revisão + HackerRank
Prática
Resolver 3 problemas no HackerRank (nível easy).
1hackerrank.com → "10 Days of JavaScript" → Day 0, 1, 2
2Commit: git commit -m "feat: week 1 javascript basics"

O Event Loop
setTimeoutCall Stack

2Prediga ANTES de rodar: console.log('1'); setTimeout(()=>console.log('2'),0); console.log('3')

Generics, Utility Types e Type Guards. Isso é o que separa um dev que "usa TS" de um que realmente entende TS.

✓DiaAulaO que fazer

Generics
function<T>
1Problema sem generics: function first(arr: any[]) {} — perde a tipagem

Utility Types
PartialPickOmit
1Partial<User> — todos os campos opcionais (para updates)
2Pick<User, 'id' | 'name'> — seleciona campos específicos

3

typeofinstanceofin

2Discriminated unions: type Shape = {kind:'circle',radius:number} | {kind:'square',side:number}
3Função getArea(shape: Shape) com switch no kind
4
Classes Tipadas
privatereadonlyabstract
1Modificadores: public, private, protected, readonly
2Shorthand: constructor(private name: string, public age: number)

5
Mapped & Conditional Types
Avançado
1Mapped type: type Optional<T> = { [K in keyof T]?: T[K] }

Semana 16 — Autenticação JWT

Segurança é responsabilidade do dev. Hash de senhas, JWT, refresh tokens e rotas protegidas.

✓DiaAulaO que fazer
1
Como JWT funciona
header.payload.sig
1Decodifique um token em jwt.io — veja header, payload, signature
2JWT NÃO é criptografado — apenas assinado. Nunca coloque senha no payload!
3Access token: curto (15min). Refresh token: longo (7 dias) e armazenado no DB
2
bcrypt + Hash de Senhas
1npm install bcryptjs
2Hash: const hash = await bcrypt.hash(password, 12)
3Compare: const isValid = await bcrypt.compare(password, hash)
4Nunca armazene senha em plain text. Use sempre salt rounds ≥ 10.
3
Register & Login
1POST /auth/register: valide email, hash senha, salve user, retorne JWT
2POST /auth/login: busque user, compare senha, gere access + refresh token
3npm install jsonwebtoken
4
Auth Middleware
1Middleware verifica Authorization: Bearer <token>
2jwt.verify(token, secret) → decodifica e adiciona user em req.user
3Aplique em rotas protegidas: router.get('/profile', authenticate, controller)
5
Refresh Token
1Armazene refresh tokens no DB (tabela RefreshToken)
2POST /auth/refresh: valide refresh token, emita novo access token
3POST /auth/logout: invalide refresh token no DB
6
Sistema de Auth Completo
Projeto
1Integre auth na API de Produtos: somente admin pode criar/editar/deletar
2Role-based: middleware checkRole(['ADMIN'])
3Teste fluxo completo: register → login → acessar rota protegida → refresh → logout
